Phase 7 · custody

Where the keys live is pluggable

Signing keys sit behind one abstraction. The demo keeps them on disk; the desktop build keeps them in the operating system's keyring — and one command moves an existing workspace across.

Demo · default

File vault

Keys as keys/author-N.key on disk. Single-operator, loopback only — not production custody.

→

Desktop · STUDIO_CUSTODY=keyring

OS keyring

Keychain · Credential Manager · Secret Service. Nothing sensitive on disk.

migrate to the OS keyring
$ ls studio-data/keys
author-1.key  author-1.master.key  author-2.key  author-2.master.key  author-3.key  author-3.master.key

$ aion-studio-api import-keys   # desktop build: --features keyring
validated 3 author(s) for keyring import: [1, 2, 3]
imported 3 author(s) into the OS keyring

The keys validate against the registry before they move — a mismatch is refused. Policies, signatures, and verification are unchanged. Only custody moves.
